# S04: Blog Reading Experience

**Goal:** Enhance blog posts with tag navigation, related posts, share buttons, code copy buttons, and a sticky TOC with scroll-spy for long posts.
**Demo:** Navigate to `/blog/tag/astro/` to see filtered posts. Open a long post (>1000 words) — sticky TOC sidebar on desktop highlights active heading during scroll. Related posts appear at bottom. Share buttons link to X/LinkedIn/Dev.to. Code blocks have copy-to-clipboard button.

## Must-Haves

- Tag archive pages at `/blog/tag/[tag]/` with pagination, reusing BlogCard
- Tag pills in BlogCard and BlogPost link to their archive page
- Related posts section at bottom of posts with shared-tag matching (graceful empty state)
- Share buttons for X, LinkedIn, Dev.to on each post (URL-based, no third-party JS)
- Copy button on code blocks with clipboard API and "Copied!" feedback
- Sticky TOC sidebar on desktop with scroll-spy (IntersectionObserver), only for posts ≥5 min read (~1000 words)
- Two-column layout (content + TOC) on desktop for long posts; single-column for short posts and mobile
- At least one sample post >1000 words to exercise the TOC
- `scroll-margin-top` on headings so anchor clicks don't hide behind sticky header

## Observability / Diagnostics

- **Tag archive routing:** Verify tag archive pages are generated by checking `dist/blog/tag/*/index.html` — each unique tag must produce at least one page. Missing pages indicate a `getStaticPaths` enumeration gap.
- **Tag pill link integrity:** `grep -r 'href="/blog/tag/' dist/blog/` confirms all tag pills resolve to valid archive routes. Broken links indicate tag name mismatch between content and route params.
- **TOC rendering logic:** Long posts (≥5 min read) must contain `<nav` with TOC markup in their `dist/` HTML; short posts must not. `grep -l 'table-of-contents\|toc' dist/blog/*/index.html` surfaces which posts have TOCs.
- **Scroll-spy runtime:** IntersectionObserver errors surface in browser console. Check `browser_get_console_logs` after scrolling a long post.
- **Copy button feedback:** `navigator.clipboard.writeText` failures surface as console errors on insecure origins. Test on `localhost` (secure context).
- **Share button URLs:** Share URLs are constructed from `Astro.url` — verify correct encoding by inspecting `href` attributes in build output. Malformed URLs show as 404s on the target platform.
- **Failure visibility:** Build failures from missing imports, bad `getStaticPaths` returns, or template errors are surfaced by `npm run build` stderr. The `verify-s04.sh` script reports per-check PASS/FAIL with exit code 1 on any failure.
